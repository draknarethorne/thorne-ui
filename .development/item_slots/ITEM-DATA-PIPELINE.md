# Item Data Pipeline

Maintainer: Draknare Thorne

Extracts EverQuest item stats from the Quarm database dump and generates
class-specific icon recommendations for equipment slot art.

## Directory Layout

```
.tmp/                          ← Raw source data (gitignored)
  quarm_*.sql                    SQL dump (~200 MB, external download)
  *.sql                          Supporting SQL files (drop_system, etc.)
  quarm-db.tar.gz                Original archive
  *.py                           Superseded prototype scripts (safe to delete)

.bin/                          ← Production scripts (version-controlled)
  extract_eq_items.py            Step 1: SQL → CSV + JSON
  build_slot_reference.py        Step 2: CSV → archetype/slot icon reference
  pick_class_icons.py            Step 3: CSV → class-specific scored icon picks + HTML

.cache/                        ← Generated outputs (gitignored, regenerable)
  eq_items.csv                   26,971 items with stats + dragitem mapping
  eq_items.json                  Same data, JSON format
  slot_icon_reference.csv        Archetype → slot → icon summary
  slot_icon_reference.json       Structured archetype/slot/icon reference
  class_icon_picks.csv           Best icon per class + slot (flat)
  class_icon_picks.json          Ranked icons per class + slot (structured)
  class_icon_picks.html          Visual verification grid with PNG thumbnails
```

## Prerequisites

1. **Quarm SQL dump** in `.tmp/`:
   - Source: `SecretsOTheP/EQMacEmu` → `utils/sql/database_full/`
   - Downloaded file: `quarm-db.tar.gz`
   - Extract to `.tmp/` — the main file is `quarm_YYYY-MM-DD-HHMM.sql` (~200 MB)
   - The script auto-discovers any `quarm_*.sql` file over 10 MB

2. **Python 3.10+** — no external packages required (stdlib only)

3. **Icon PNGs** (for HTML visual verification):
   - Located at `thorne_drak/Options/Slots/.Master/.Items/.cache/dragitemN/rRcC.png`
   - Generated by `.bin/generate_catalog.py --png`
   - 36 PNGs per dragitem file (r1c1 through r6c6, 1-based)
   - dragitem1–34 available

## Running the Pipeline

Run from the project root (`C:\Thorne-UI`):

```bash
# Step 1: Extract items from SQL dump
python .bin/extract_eq_items.py

# Step 2: Build archetype → slot → icon reference
python .bin/build_slot_reference.py

# Step 3: Generate class-specific icon picks + HTML
python .bin/pick_class_icons.py
```

**Order matters** — each step depends on the previous step's output.

If you only change scoring logic in `pick_class_icons.py`, you can re-run
just Step 3. If the SQL dump or extraction logic changes, re-run all three.

### Windows Note

If you get `UnicodeEncodeError` on Windows, prefix with:
```bash
PYTHONIOENCODING=utf-8 python .bin/extract_eq_items.py
```

## Icon Mapping Formula

EQ icon numbers map to dragitem sprite sheets using this formula:

```python
adjusted = icon - 500           # EQ icons start at 500
file     = adjusted // 36 + 1   # 36 icons per dragitem file
cell     = adjusted % 36        # Position within the 6×6 grid
row      = cell % 6 + 1         # 1-based, col-major ordering
col      = cell // 6 + 1        # 1-based, col-major ordering
```

**Key facts:**
- Offset of 500 (icon 500 = dragitem1 r1c1)
- Col-major ordering (columns fill first, then rows)
- 1-based row/col (r1c1 through r6c6)
- Each dragitem file holds a 6×6 grid = 36 icons
- Verified against: Cloth Cap (639→di4 r2c6), Bone Necklace (502→di1 r3c1),
  Plague Knight Helm (506→di1 r1c2), Lightstone (507→di1 r2c2)

## Script Details

### extract_eq_items.py

**Input:** `.tmp/quarm_*.sql` (Quarm database dump)
**Output:** `.cache/eq_items.csv`, `.cache/eq_items.json`

Parses `INSERT INTO items VALUES(...)` statements. Extracts 159 columns per
item including stats (STR, STA, AGI, DEX, INT, WIS, CHA, HP, Mana, AC, resists),
class/slot bitmasks, icon numbers, and derived dragitem mapping fields.

### build_slot_reference.py

**Input:** `.cache/eq_items.csv`
**Output:** `.cache/slot_icon_reference.json`, `.cache/slot_icon_reference.csv`

Groups items by archetype (Melee/Hybrid/Priest/Caster) and equipment slot,
counting which dragitem icons appear and how frequently. Useful for seeing the
full universe of icons available per slot before narrowing to class-specific picks.

### pick_class_icons.py

**Input:** `.cache/eq_items.csv`
**Output:** `.cache/class_icon_picks.json`, `.cache/class_icon_picks.csv`, `.cache/class_icon_picks.html`

Scores icons for 9 class groupings (Melee, Hybrid, Priest, Caster, Bard, Druid,
Monk, Necro, Ranger) using stat-priority weights. Scoring includes:

- **Stat score** — weighted average of item stats matching class priorities
- **Count bonus** — rewards well-represented icons (capped at 20 items × 0.5)
- **Exclusivity bonus** — rewards icons unique to a class group (30% of stat score)
- **Min-2-item preference** — icons with 2+ items rank above single-item outliers
- **GM filter** — items with HP >500, Mana >500, or AC >300 are excluded

The HTML output includes inline PNG thumbnails for visual verification of picks.

## Maintenance

### When to re-run

| Change | Re-run |
|--------|--------|
| New SQL dump | All 3 steps |
| Fix extraction logic | All 3 steps |
| Change archetype definitions | Steps 2 + 3 |
| Change class scoring weights | Step 3 only |
| Change icon mapping formula | All 3 steps |

### Cleaning up .tmp

The `.tmp/` directory contains old prototype scripts from the development
process. These are safe to delete — all production logic lives in `.bin/`.
Only the SQL dump files (`quarm_*.sql`, supporting `.sql` files, and
`quarm-db.tar.gz`) are needed as source data.

### Adding a new class grouping

1. Edit `pick_class_icons.py` → `CLASS_PROFILES` dict
2. Add the class name and stat weights
3. Re-run Step 3
4. Check the HTML output for reasonable icon differentiation

### Verifying icon mappings

Open `.cache/class_icon_picks.html` in a browser. Each icon pick shows:
- Inline PNG thumbnail (from dragitem cache)
- Icon number and dragitem coordinates
- Score breakdown, top stats, and example item names
- Class divergence overview at the top
